# Plan Maestro: Detección y Conteo de Moscas (YOLO26 + SAHI + SAM 3)
**Referencia Técnica:** `@contextAI/ContextoMoscas.pdf`

Este documento detalla el pipeline de ejecución para el monitoreo automatizado de *Haematobia irritans*. El sistema debe operar bajo restricciones de hardware (<4GB RAM, <100ms latencia) y superar la asimetría de escala (Vaca 2.5m vs. Mosca 4mm).

---

## 🏗️ 1. Arquitectura del Pipeline (Fase de Inferencia)
Para cumplir con los tiempos de respuesta en el servidor Cratos, se divide la ejecución en dos estadios:

### Etapa 1: Aislamiento del Bovino (Macro-Escala)
*   **Modelo:** **YOLO26-Nano/Small** (Entrenado en V16/V17).
*   **Herramientas:** Inferencia estándar Ultralytics.
*   **Proceso:** Detectar la clase `Cow` o `Back` y extraer las coordenadas de la ROI.
*   **Output:** Recorte (Crop) de alta resolución del lomo guardado en memoria RAM.

### Etapa 2: Conteo Hiper-Asistido (Micro-Escala)
*   **Modelo:** **YOLO26-Medium (NMS-Free)**.
*   **Técnica:** **SAHI (Slicing Aided Hyper Inference)**.
*   **Por qué YOLO26 NMS-Free:** Según `ContextoMoscas.pdf` (Pág. 12), elimina el cuello de botella del NMS tradicional en nubes densas de moscas, permitiendo una predicción "uno a uno" directa y veloz.
*   **Funcionamiento:** SAHI divide la ROI en sub-rebanadas (slices de 512x512), las pasa por YOLO26 y luego fusiona las detecciones.

---

## 🧪 2. Data Engine: Fábrica de Etiquetas (Fase de Preparación)
Dado que SAM 3 es demasiado pesado para el tiempo real, se usa exclusivamente para generar el **Ground Truth** perfecto de forma automática.

### Fase A: Discovery (Detección de Candidatos por Contraste)
*   **Herramienta:** **OpenCV (Visión Artificial Tradicional)**.
*   **Proceso:** Dado que no existe aún un modelo de moscas, se utiliza el contraste natural (mosca oscura sobre piel clara):
    1. Conversión del recorte del lomo a escala de grises.
    2. Aplicación de umbral adaptativo (`Adaptive Thresholding`) para resaltar manchas oscuras.
    3. Detección de contornos y cálculo de centroides (coordenadas X, Y).
*   **Objetivo:** Generar los "puntos" (prompts) que SAM 3 requiere para activarse.
*   **Razón Técnica:** Evita el fallo de usar modelos de vacas (V17) para detectar objetos sub-píxel.

### Fase B: Refinement (Segmentación con SAM 3)
*   **Herramienta:** **Segment Anything Model 3 (Meta)**.
*   **Proceso:** Recibe los puntos de la Fase A y dibuja el polígono exacto del exoesqueleto de la mosca.
*   **Ventaja:** Ignora texturas del pelo y sombras, logrando precisión de "grado científico".

### Fase C: Conversion (YOLO Format)
*   **Herramienta:** Script de transformación (Python).
*   **Proceso:** Convertir las máscaras de SAM 3 en Bounding Boxes estandarizados `.txt` para entrenar el modelo final.

## ⚙️ 3. Configuración de Entrenamiento Especializado
El entrenamiento de YOLO26 seguirá las directrices biológicas del PDF (Pág. 13-15):

1.  **STAL (Small-Target Aware Label Assignment):** Asegura que los objetos de 4px generen gradientes de aprendizaje fuertes.
2.  **Restricción Biológica de Aumento:** 
    *   **AutoAugment:** Rotaciones limitadas a **±45°**. 
    *   **Razón:** Las moscas siempre orientan su cabeza hacia el suelo por gravedad; rotaciones de 180° crearían datos físicamente imposibles.
3.  **GridMask:** Borrado de mallas para obligar al modelo a aprender la forma de las alas y patas, no solo el brillo del lomo.
