# 🐄 Plan de Etiquetado Jerárquico con SAM 3 (Zero-Shot PCS)

Este documento detalla la estrategia para la generación automatizada de etiquetas de alta precisión para el dataset de 173,956 imágenes, utilizando el modelo fundacional **Segment Anything 3 (SAM 3)**.

---

## 🎯 1. Objetivos del Etiquetado
1. **Detección y Segmentación de Ganado:** Obtener la silueta exacta y el cuadro delimitador de cada bovino.
2. **Identificación de Partes Clave (Key Parts):** Siguiendo la metodología del paper *Shao et al. (2023)*, detectar: **Cuerpo, Cabeza, Patas y Cola**.
3. **Filtrado de Humanos:** Identificar operarios para evitar falsos positivos y asegurar la pureza del dataset.
4. **Formato Dual:** Generar archivos compatibles con modelos YOLO de **Detección** (Bounding Boxes) y **Segmentación** (Polígonos).

---

## 🛠️ 2. Especificaciones Técnicas
- **Modelo:** SAM 3 (Weights: `sam3.pt` - 3.45 GB).
- **Entorno:** `tesis_vacas` (Python 3.10, PyTorch, SAM3-Meta).
- **Estrategia Multi-GPU:** Uso de **4 GPUs (IDs 1, 2, 3, 4)**.
- **Técnica:** Promptable Concept Segmentation (PCS) mediante lenguaje natural.

---

## 🏷️ 3. Jerarquía de Clases y Prompts
| ID Clase | Prompt | Descripción |
| :--- | :--- | :--- |
| **0** | `"cow"` | Cuerpo completo del animal (Vaca/Toro/Ternero). |
| **1** | `"cow head"` | Cabeza del bovino. |
| **2** | `"cow leg"` | Patas visibles del animal. |
| **3** | `"cow tail"` | Cola del bovino. |
| **-** | `"person"` | **CLASE NEGATIVA**: Se usa para descarte automático si Conf > 0.5. |

---

## 📁 4. Estructura de Salida (Producción Final)
Las etiquetas se organizarán en `/data/estudiantes/vacas/dataset_curado/train/labels/` con la siguiente estructura:

```text
labels/
├── full_cow/
│   ├── bbox/ (Formato YOLO Detección: class x_c y_c w h - Solo Clase 0)
│   └── seg/  (Formato YOLO Segmentación: class x1 y1 x2 y2... - Solo Clase 0)
└── key_parts/
    ├── bbox/ (Incluye las IDs 1, 2 y 3: head, legs, tail)
    └── seg/  (Segmentación de las partes clave IDs 1, 2, 3)
```

---

## 🛡️ 5. Robustez y Resiliencia (Mecanismo de Checkpoint)
Para garantizar que el proceso no se pierda ante interrupciones:
1. **Registro de Progreso:** Archivos `logs/progreso_sam3_gpuX.csv` guardan las imágenes completadas.
2. **Reanudación Automática:** El script salta automáticamente archivos ya registrados.
3. **Muestreo QC:** Se genera 1 visualización por cada 500 imágenes en `verificacion_etiquetas/produccion_sam3/`.

---

## 🚀 6. Guía de Ejecución Multi-GPU (Segundo Plano)
El dataset se divide en 4 fragmentos. Comandos para lanzar:

```bash
nohup /opt/anaconda3/envs/tesis_vacas/bin/python 10.sam3_segmentation.py --gpu 1 --shard 0 > logs/segmentacion_gpu1.log 2>&1 &
nohup /opt/anaconda3/envs/tesis_vacas/bin/python 10.sam3_segmentation.py --gpu 2 --shard 1 > logs/segmentacion_gpu2.log 2>&1 &
nohup /opt/anaconda3/envs/tesis_vacas/bin/python 10.sam3_segmentation.py --gpu 3 --shard 2 > logs/segmentacion_gpu3.log 2>&1 &
nohup /opt/anaconda3/envs/tesis_vacas/bin/python 10.sam3_segmentation.py --gpu 4 --shard 3 > logs/segmentacion_gpu4.log 2>&1 &
```

## 📊 7. Monitoreo del Progreso
Para saber si todo va bien mientras los scripts corren en segundo plano:

1. **Ver logs en tiempo real:**
   ```bash
   tail -f logs/segmentacion_gpu1.log
   ```
   *(Cambia el 1 por 2, 3 o 4 para ver las otras GPUs).*

2. **Verificar que los procesos siguen vivos:**
   ```bash
   ps aux | grep 10.sam3_segmentation.py
   ```

3. **Verificar uso de GPU:**
   ```bash

   ```
