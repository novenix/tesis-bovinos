# Plan de Entrenamiento: Bovinos Segmentación V2 (YOLO26)

Este documento sirve como hoja de ruta para la evolución del modelo tras los hallazgos del Script 15. **Nota:** Se ha decidido saltar a **YOLO26** para estar en el SOTA absoluto de 2026.

## 📌 Estado Actual (Aprendizajes de V15 con YOLO11)
*   **Detección de Cajas (Boxes):** Sólida (0.43 mAP50).
*   **Segmentación (Masks):** Colapsó al final de las 100 épocas (mAP ~0.0005). El modelo YOLO11 "olvidó" los polígonos.
*   **Diagnóstico Vértices:** Etiquetas con hasta **2010 vértices**. Inmanejable para un modelo Nano.
*   **Estructura:** El entrenamiento anterior es **`train1`**. El nuevo será **`train2`**.

## 🚀 Estrategias para `16.entrenar_bovinos_puro_2.py` (YOLO26)

### 1. Infraestructura y Seguridad de Hardware (Cratos)
*   **Vigilancia de Recursos (RAM + GPU):** Monitoreo de RAM, VRAM y Temperatura de las 5 GPUs (3,4,5,6,7).
*   **Distribución de Carga:** `os.environ["CUDA_VISIBLE_DEVICES"] = "3,4,5,6,7"`, `batch_size = 80`.

### 2. Actualización de Modelo y Optimizador
*   **YOLO26-seg:** Arquitectura NMS-Free.
*   **MuSGD (SGD + Muon):** Estabilizador de polígonos.

### 3. Aumentos Inteligentes "On-The-Fly" (GridMask)
*   **Albumentations RAM:** GridMask y AutoAugment en memoria.

### 4. Reequilibrio y Congelación Estructural
*   **Pesos de Loss:** `mask=25.0`, `box=5.0`.
*   **Backbone Freezing:** `freeze=10`.
*   **Erasing:** Reducir a 0.15.

### 5. Auditoría y Simplificación No-Destructiva de Etiquetas
*   **Douglas-Peucker:** Reducir polígonos a ~100 vértices en carpeta espejo `/dataset_simplified_labels/`. **NO se tocan originales**.

### 6. Validación Visual "Human-in-the-Loop"
*   **Muestreo cada 10 épocas:** 5 fotos aleatorias con máscaras predichas en `/verificacion_manual/entrenamiento_v16/`.

### 7. Telemetría y Métricas Exhaustivas del Modelo (Paper-Ready)
*   mAP50/95, F1-Score, IoU, Matriz de Confusión y Benchmarking de velocidad (ms).
